## 朝食
- 報告担当2人
- issue用ブランチへのマージまで行う(staging,masterへのマージは行わない)※アプリ内書店のようにしばらく開発するパターン

## 昼食
- 報告担当1人
- レビュー担当者は、issue用ブランチへのプルリクマージ前に最低1度レビューで修正依頼を出す
- staging,masterへのマージまで行う※即時対応、即時リリースのパターン
